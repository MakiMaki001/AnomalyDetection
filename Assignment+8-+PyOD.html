3rd set of variables¶

  1. Making key ratios from the cost based variables and total discharge. We will look at the ratio between covered charges and medicare to total payments. In addition, we identify the total payments per discharge, medicare payments per discharge, and covered payments per discharge.

In regards to the features created for the 2nd and 3rd grouping (Difference and Ratio based), it is unclear if the approach that was done is the best way using pandas. The approach relied on the .apply function to create a newly calculated variable (similar to 'Mutate' in R). This often times created a warning titled SettingWithCopyWarning. We will need to analyze this in more detail to see what the best approach will be to mutate new models using python.

##Adding 3rd set of features- ratio based variables

df_ratio = df[['DRG_Definition', 'Provider_Name', 'Provider_State','Total_Discharges', 
                         'Average_Covered_Charges', 
                    'Average_Total_Payments', 'Average_Medicare_Payments']]

df_ratio['Medicare_Percentage'] = df.groupby(['DRG_Definition','Provider_State', 'Provider_Name'], 
  group_keys=False).apply(lambda g: g.Average_Medicare_Payments/g.Average_Total_Payments.sum())

df_ratio['Covered_Percentage'] = df_ratio.groupby(['DRG_Definition','Provider_State', 'Provider_Name'], 
  group_keys=False).apply(lambda g: g.Average_Covered_Charges/g.Average_Total_Payments.sum())

df_ratio['Discharge_Percentage'] = df_ratio.groupby(['DRG_Definition','Provider_State', 'Provider_Name'], 
  group_keys=False).apply(lambda g: g.Total_Discharges/g.Average_Total_Payments.sum())

df_ratio['Medicare_Per_Discharge'] = df_ratio.groupby(['DRG_Definition','Provider_State', 'Provider_Name'], 
  group_keys=False).apply(lambda g: g.Average_Medicare_Payments/g.Total_Discharges.sum())

df_ratio['TotPayment_Per_Discharge'] = df_ratio.groupby(['DRG_Definition','Provider_State', 'Provider_Name'], 
  group_keys=False).apply(lambda g: g.Average_Total_Payments/g.Total_Discharges.sum())

df_ratio['Covered_Per_Discharge'] = df_ratio.groupby(['DRG_Definition','Provider_State', 'Provider_Name'], 
  group_keys=False).apply(lambda g: g.Average_Covered_Charges/g.Total_Discharges.sum())

KNN Model¶

The K Nearest Neighbor function is an unsupervised approach which creates clusters in calculation to the proximity of underlying data points. The Generallly considered largest weakness with the KNN approach is that the user must specify how many clusters should be counted from the dataset. This is a particularly large weakness for this dataset since there are so many observations within it. The model calculates points by measuring the mean distance of all points around them. Once it has identified the center value (where the mean is equadistant from all other points in the cluster) within the number of clusters that was specified by the user, the model is completed and the clusters identified.

Therefore, the unsupervised approach creates groupings (or clusters) of the underlying dataset in line with the center points and the related data points that are nearest to that calculated center. This is a useful model for a data scientist to iterate continuously until they find the best fits (in regards to the number of clusters and data fit), which can then be applied to more complicated models for outlier analysis.

Meanshift Model¶

The meanshift model measures the shortest distances between points using eucledian distance to identify nearest pints. As opposed to other auto clustering methods (such as DBScan, for example), meanshift moves along the dense data point areas of the dataset in order to identify which items belong to a cluster. It is considered a 'hill climbing' algorithim since it begins to calculate the clusters starting at a random point and then calculating which are the nearest features within the dataset. Since it looks for datapoints which are near each other, the model naturally moves towards the denser regions of where the data lies in the matrix. Once density within a window of points decreases, the model completes that cluster and moves to repeat this process on other, non covered data points in the set.
